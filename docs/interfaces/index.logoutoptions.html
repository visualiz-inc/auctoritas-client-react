Optional client_id

client_id: string
  • Defined in node_modules/@auctoritas/client-vanilla/dist/typings/global.d.ts:363

The client_id of your application.

If this property is not set, then the client_id that was used during initialization of the SDK is sent to the logout endpoint.

If this property is set to null, then no client ID value is sent to the logout endpoint.

Optional federated

federated: boolean
  • Defined in node_modules/@auctoritas/client-vanilla/dist/typings/global.d.ts:371

When supported by the upstream identity provider, forces the user to logout of their identity provider and from Auctoritas. This option cannot be specified along with the localOnly option. Read more about how federated logout works at Auctoritas

Optional localOnly

localOnly: boolean
  • Defined in node_modules/@auctoritas/client-vanilla/dist/typings/global.d.ts:378

When true, this skips the request to the logout endpoint on the authorization server, effectively performing a "local" logout of the application. No redirect should take place, you should update local logged in state. This option cannot be specified along with the federated option.

Optional returnTo

returnTo: string
  • Defined in node_modules/@auctoritas/client-vanilla/dist/typings/global.d.ts:353

The URL where Auctoritas will redirect your browser to after the logout.

Note: If the client_id parameter is included, the returnTo URL that is provided must be listed in the Application's "Allowed Logout URLs" in the Auctoritas dashboard. However, if the client_id parameter is not included, the returnTo URL must be listed in the "Allowed Logout URLs" at the account level in the Auctoritas dashboard.
