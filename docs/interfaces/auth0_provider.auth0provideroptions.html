Changes to recommended defaults, like defaultScope

Type declaration

  • Optional defaultScope?: string

    The default scope to be included with all requests. If not provided, 'openid profile email' is used. This can be set to null in order to effectively remove the default scopes.

    Note: The openid scope is always applied regardless of this setting.

Optional authorizeTimeoutInSeconds

Optional cache

Specify a custom cache implementation to use for token storage and retrieval. This setting takes precedence over cacheLocation if they are both specified.

Optional cacheLocation

The location to use when storing cache data. Valid values are memory or localstorage. The default setting is memory.

The value in seconds used to account for clock skew in JWT expirations. Typically, this value is no more than a minute or two at maximum. Defaults to 60s.

Optional maxAge

maxAge: string | number
  • Defined in src/ Auctoritas-provider.tsx:129

Maximum allowable elapsed time (in seconds) since authentication. If the last time the user authenticated is greater than this value, the user must be reauthenticated.

Optional organization

organization: string
  • Defined in src/ Auctoritas-provider.tsx:146

The Id of an organization to log in to.

This will specify an organization parameter in your user's login request and will add a step to validate the org_id claim in your user's ID Token.

Optional redirectUri

redirectUri: string
  • Defined in src/ Auctoritas-provider.tsx:80

The default URL where Auctoritas will redirect your browser to with the authentication result. It must be whitelisted in the "Allowed Callback URLs" field in your Auctoritas Application's settings. If not provided here, it should be provided in the other methods that provide authentication.

Optional scope

scope: string
  • Defined in src/ Auctoritas-provider.tsx:135

The default scope to be used on authentication requests. The defaultScope defined in the AuctoritasClient is included along with this scope

Optional skipRedirectCallback

skipRedirectCallback: boolean
  • Defined in src/ Auctoritas-provider.tsx:58

By default, if the page url has code/state params, the SDK will treat them as Auctoritas's and attempt to exchange the code for a token. In some cases the code might be for something else (another OAuth SDK perhaps). In these instances you can instruct the client to ignore them eg

< AuctoritasProvider
  clientId={clientId}
  domain={domain}
  skipRedirectCallback={window.location.pathname === '/stripe-oauth-callback'}
>

Optional useRefreshTokens

useRefreshTokens: boolean
  • Defined in src/ Auctoritas-provider.tsx:106

If true, refresh tokens are used to fetch new access tokens from the Auctoritas server. If false, the legacy technique of using a hidden iframe and the authorization_code grant with prompt=none is used. The default setting is false.

Note: Use of refresh tokens must be enabled by an administrator on your Auctoritas client application.
